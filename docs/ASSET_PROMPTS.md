# 美工生成指南 · ASSET_PROMPTS

> 给图像生成模型（Midjourney / SDXL / DALL·E / 即梦 / 通义万相 等）的 prompt 清单。
> 每张图都已经在 `event.json` 里登记好了文件名，生成完按路径放进去即可。

## 通用风格指南（每张图都先粘到 prompt 前面）

**画风（中文 prompt 推荐）**

```
中国唐代水墨设定画风，半写实，柔和水彩晕染叠加细线工笔，唐代服饰与建筑严格考证，构图电影感，温暖典雅的暮色调，柔光，无人物（或人物按描述），构图中央有大片空白便于放置交互点，避免现代元素，避免文字水印，16:9 横构图
```

**画风（英文 prompt 推荐 — 适合 MJ / SDXL）**

```
Tang Dynasty Chinese ink-wash + watercolor wash painting, semi-realistic, soft brushwork with fine line detail, historically accurate Tang costumes and architecture, cinematic composition, warm dusk palette with gentle light, soft mist, leave generous empty space in mid-foreground for interactive elements, no modern elements, no text or watermark, 16:9 horizontal
```

**反面 prompt（negative，所有图都加）**

```
no text, no watermark, no signature, no modern buildings, no anachronisms, no chibi, no anime style, no harsh contrast, no oversaturation, avoid clutter in foreground
```

**输出规格**
- 背景：**1920×1080**（或 16:9 同比），PNG
- 立绘：**768×1024** 透明背景 PNG（人物全身或半身居中，下半身可裁切）
- 文件名与下表完全一致，放进对应的 `public/assets/...` 路径

**重要约束**
- 背景里不要出现要交互的 NPC（NPC 用立绘独立放置）—— 除非该 NPC 在 SCREENPLAY 中明确标记需用 `bubbleMode`（已画在背景里，头顶会浮气泡）
- 整套画面的色调与笔触需保持一致（建议同一个 seed/style reference 跑完整批）

## 四、生成顺序建议

1. **先做主立绘**（gaoshi / libai / yanwu / wife / mother）—— 这几个出场多。
2. **再做战乱组**（refugee_woman / refugee_old / rebel_officer / messenger / old_woman / officer_night）—— 风格统一最重要。
3. **再做背景**——按时间顺序，736 → 770。每个事件 4-6 张一起出，便于检查同一事件的调子一致。
4. 每张图生成后，把文件放到 SCREENPLAY / 本文档列出的对应路径，运行 `npm run dev`，进游戏里看一眼比例与色调，发现不协调当场重抽。
5. **批次生成 tip**：同一事件的所有图最好用同一个 seed / style reference / 参考图。这样一组里光照、笔触、色温能保持一致。
